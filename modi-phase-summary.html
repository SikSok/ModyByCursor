📝 后续工作展望与建议

在保证「换机拉取即可跑」的前提下，建议按以下顺序推进。

优先级 建议项 说明
P0 服务端接口与数据库 按 modi-api-design / modi-database-design 打通注册、登录、定位上报、附近司机等接口；确保 user-app / driver-app 能连真实后端。
P0 司机端核心流程 定位权限、定时上报、接客状态切换与接口对接；与进度表第三阶段任务对齐。
P1 用户端核心流程 地图、附近司机展示、选司机等；与进度表第四阶段任务对齐。
P1 管理端与审核 司机审核、用户列表、数据统计；与进度表第五阶段任务对齐。
P2 联调与上架准备 四端联调、iOS 配置与打包、提审准备；与进度表第六阶段任务对齐。
P2 可选：RN 升级 当前 0.74 可稳定使用；若后续需要新特性，再按官方升级助手做一次 RN 升级并全量回归。
换机开发提醒
新电脑拉取后：各目录执行 npm install；为 user-app / driver-app 的 android 目录补全 local.properties（sdk.dir=本机Android_SDK路径）。详见根目录 README.md。
