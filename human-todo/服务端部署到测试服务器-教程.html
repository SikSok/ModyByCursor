🖥️ 服务端部署到测试服务器 - 教程（需人待办）

摩迪项目 · 短信用阿里云，服务器建议同选阿里云，便于内网与管控

一、服务器选型：用阿里云可以吗？

建议 可以。短信已经用阿里云，服务器也用阿里云的好处：

  • 同一账号下，控制台、账单、工单一处搞定
  • 若以后短信和服务器要内网互通（如同地域），可直接用内网地址，省流量、更安全
  • ECS 和 RDS（云数据库）同地域延迟低，若以后 MySQL 迁到 RDS 也方便

其他云（腾讯云、华为云等）也完全可以，按你习惯选即可；本教程以阿里云 ECS 为例。

二、服务器「种类」怎么选？

阿里云上跑 Node 服务，常见几种：

  • ECS 云服务器：最通用，自己装系统、装 Node、装 MySQL，灵活。适合长期、要自己控环境。
  • 轻量应用服务器：比 ECS 简单、便宜，带宽和流量包更直观，适合小站、测试环境。也能装 Node + MySQL。
  • 函数计算 FC / 容器服务：不用自己管服务器，按请求计费或按容器跑。适合无状态 API，但本项目用 MySQL、有状态，建议先不选。
推荐 测试阶段：轻量应用服务器（便宜、够用）或 ECS 最低配（如 1 核 2G）。正式运营再按流量和稳定性升级。

三、服务器要装什么？

摩迪服务端是 Node.js + MySQL，所以服务器上需要：

  • Node.js：跑 npm run build 后的 node dist/index.js（或用 pm2）。建议 Node 18+ LTS。
  • MySQL：本机装 MySQL 8，或以后改用阿里云 RDS。先在服务器装 MySQL 即可：创建库 mody_db，把 .env 里 DB 配置指向服务器本机或 RDS。
  • （可选）Nginx：做反向代理、绑域名、配 HTTPS；若暂时只跑测试 IP:端口，可先不装。

是的，服务器上要装 MySQL（或使用云数据库 RDS）。当前项目是「自建 MySQL」，所以要么在 ECS/轻量里装 MySQL，要么买 RDS 实例并把 DB_HOST 改成 RDS 地址。

四、部署步骤概要（需你本机/服务器操作）

1. 买服务器

  • 登录阿里云 → 轻量应用服务器 或 ECS → 选地域（如华东）、镜像选 Linux（Ubuntu 22.04 或 CentOS 7/8 均可）→ 购买。
  • 记下公网 IP，用 SSH 或控制台终端登录。

2. 装 Node.js

  • Ubuntu 示例：curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -，再 sudo apt install -y nodejs。
  • 或从 Node 官网下二进制包放到 /usr/local 并配 PATH。

3. 装 MySQL

  • Ubuntu：sudo apt update && sudo apt install -y mysql-server，sudo mysql_secure_installation 设 root 密码。
  • 登录 MySQL：CREATE DATABASE mody_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;，并创建一用户给应用用（或直接用 root 先测）。

4. 上传代码并运行

  • 本机打包 server（可排除 node_modules），用 scp / 宝塔 / Git 传到服务器。
  • 服务器上：cd server && npm install --production && npm run build。
  • 在服务器上建 .env，填 DB_HOST=127.0.0.1、DB_PASSWORD=...、JWT_SECRET=... 等。
  • 运行：node dist/index.js 或 npx pm2 start dist/index.js --name mody-api。

5. 安全组 / 防火墙

  • 阿里云控制台里给实例放行端口：如 3000（API）、22（SSH）。MySQL 若只本机访问可不放行 3306。

五、小结

  • 服务器：建议阿里云 ECS 或轻量，和短信同一家；测试阶段 1 核 2G 即可。
  • 要装的：Node.js、MySQL（或改用 RDS）；Nginx 可选。
  • 部署：代码上传 → npm install --production && npm run build → 配 .env → node dist/index.js 或 pm2。

💡 具体命令以你选的系统为准，遇报错可查阿里云文档或把报错贴给 AI 排查
