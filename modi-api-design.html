RESTful API

摩迪 App · 接口清单（mody-app + 管理端 Web）

Base URL：/api · 统一返回：{ success, message, data } · C 端调用方为 mody-app（乘客/司机身份）

1) 健康检查

GET /health
服务运行状态

2) 验证码（短信）

POST /verification-codes/send
发送验证码（register/login）
Body说明
{ phone, type } type ∈ register | login；开发环境返回 code 便于联调（生产不返回）

3) mody-app 乘客身份（用户接口）

认证
密码 + 验证码校验（注册）
方法Path用途Body/Query
POST/users/register注册{ phone, password, code, name? }
POST/users/login登录{ phone, password }
GET/users/profile个人信息Header: Authorization: Bearer <token>
PATCH/users/profile更新头像/昵称{ name?, avatar? }
地图/附近司机
查询 10km 内已审核且可接客司机（基于 driver_locations）
方法Path用途Query
GET /users/nearby-drivers 附近司机列表 ?lat=&lng=&radius_km=10

4) mody-app 司机身份（司机接口）

认证与资料
注册需验证码；登录需审核通过（status=approved）
方法Path用途Body
POST/drivers/register注册（待审核）{ phone,password,code,name,id_card?,license_plate?,vehicle_type? }
POST/drivers/login登录（需通过审核）{ phone,password }
GET/drivers/profile司机资料Header: Authorization
PATCH/drivers/profile更新资料{ name?, avatar?, id_card?, license_plate?, vehicle_type? }
定位与接客状态
每 30 秒上报定位；is_available 控制可接客
方法Path用途Body
POST/drivers/location上报定位{ latitude, longitude, accuracy? }
PATCH/drivers/availability切换接客状态{ is_available: boolean }

5) 管理端（admin-web）

管理员认证
Web 登录后才能审核司机
方法Path用途Body
POST/admins/login登录{ username, password }
ADMIN 司机审核
drivers.status: pending → approved/rejected
方法Path用途说明
GET/admins/drivers/pending待审核列表Header: Authorization
PATCH/admins/drivers/:id/approve审核通过Header: Authorization
PATCH/admins/drivers/:id/reject审核驳回Header: Authorization
落地说明（与当前代码一致）
  • C 端：mody-app（乘客/司机身份切换，调用用户与司机接口）；服务端 server；管理端 admin-web。原 user-app、driver-app 目录暂保留。
  • 当前 server 已实现：验证码发送、用户/司机注册与登录（含验证码校验）、定位上报、附近司机查询（基于最后一次定位）、接客状态、管理员登录与司机审核、用户列表、数据统计等接口；部署见 docs/deploy-server.md。
  • 后续可扩展：司机资料上传图片（身份证/行驶证）、操作日志表等。供 Phase 6 联调与上架使用。
