摩的 App 需求文档

C 端一体化（mody-app）+ 服务端 + 管理端 · 基于 90 天开发进度梳理

产品目的与目标

摩迪（摩的）App 旨在连接乘客与摩托车司机，提供「附近司机可见、一键联系」的短途摩的出行服务。目标用户包括：需要摩的出行的乘客、提供服务的司机、以及进行审核与数据运营的管理员。产品通过服务端统一接口与数据、mody-app（C 端一体化：注册后选择身份，乘客/司机对应不同主界面，身份可在个人中心切换）、管理端司机审核与数据看板，在 90 天内完成从 0 到 1 的交付与 iOS 上架。注：原 user-app、driver-app 已整合为 mody-app，旧目录暂保留、后续再删除。

三端核心功能清单（mody-app + server + admin-web）

⚙
服务端
数据与接口中枢
  • Express + TypeScript 环境，MySQL（Sequelize），CORS、JWT 鉴权
  • 用户注册 / 登录（含短信验证码校验），司机注册 / 登录
  • 司机定位上报与存储，Haversine 计算附近司机（10 公里内）
  • 司机接客状态更新接口
  • 管理端：待审核司机列表、审核通过/驳回、用户列表、数据统计面板接口
  • 阿里云短信服务（验证码），统一错误处理格式，接口文档与测试服务器部署
📱
mody-app（C 端一体化）
乘客与司机同一 APP，身份选择与个人中心切换
  • React Native，统一注册/登录后身份选择（乘客 / 司机），不同身份进入不同主界面；个人中心可切换身份
  • 乘客身份：高德地图与当前定位、附近司机列表与 Marker（分色）、司机信息弹窗、拨打电话、个人中心与实名认证状态
  • 司机身份：司机注册/资料、定位权限与定时上报、接客状态切换、待审核提示、「我的」与退出登录
  • 验证码登录、网络异常与错误提示、边缘场景处理与地图性能优化
📊
管理端
审核与运营看板
  • React Native + Expo 项目初始化
  • 司机审核列表页（对接待审核司机接口）
  • 审核通过 / 驳回操作及接口对接
  • 数据统计面板与图表展示
  • 与服务端审核、用户列表、统计接口联调与验收
