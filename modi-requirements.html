摩的 App 需求文档

四端核心功能清单 · 基于 90 天开发进度梳理

产品目的与目标

摩迪（摩的）App 旨在连接乘客与摩托车司机，提供「附近司机可见、一键联系」的短途摩的出行服务。目标用户包括：需要摩的出行的乘客、提供服务的司机、以及进行审核与数据运营的管理员。产品通过服务端统一接口与数据、司机端实时定位与接客状态、用户端地图选司机与联系、管理端司机审核与数据看板，在 90 天内完成四端从 0 到 1 的交付与 iOS 上架。

四端核心功能清单

⚙
服务端
数据与接口中枢
  • Express + TypeScript 环境，MySQL（Sequelize），CORS、JWT 鉴权
  • 用户注册 / 登录（含短信验证码校验），司机注册 / 登录
  • 司机定位上报与存储，Haversine 计算附近司机（10 公里内）
  • 司机接客状态更新接口
  • 管理端：待审核司机列表、审核通过/驳回、用户列表、数据统计面板接口
  • 阿里云短信服务（验证码），统一错误处理格式，接口文档与测试服务器部署
🛵
司机端
司机接单与状态
  • React Native + Expo，导航栈（注册 / 登录 / 首页）
  • 司机注册 UI 与表单校验（react-hook-form），登录（含验证码）
  • 定位权限申请、获取并显示当前定位，每 30 秒定时上报定位
  • 接客状态切换（空闲 / 接客）及状态同步接口
  • 「我的」页面与待审核状态提示，退出登录
  • 网络异常与错误提示，定位上报防抖与边缘场景处理
📱
用户端
乘客找司机与联系
  • React Native + Expo，用户注册 / 登录（含验证码）
  • 高德地图集成与当前定位，获取附近司机列表接口
  • 地图 Marker 标记司机（分色区分状态），司机信息弹窗与点击交互
  • 拨打电话功能，个人中心与实名认证状态展示
  • 无司机 / 无定位等边缘场景处理，地图性能优化（缓存 / 懒加载）与加载动画
📊
管理端
审核与运营看板
  • React Native + Expo 项目初始化
  • 司机审核列表页（对接待审核司机接口）
  • 审核通过 / 驳回操作及接口对接
  • 数据统计面板与图表展示
  • 与服务端审核、用户列表、统计接口联调与验收
